# Chrome Web Store 公開手順

本ドキュメントは X OFF を Chrome Web Store に公開するための手順をまとめたものです。初回公開と継続更新の両方をカバーします。

## 1. 事前準備（一度だけ必要）

### 1-1. Chrome Web Store Developer 登録

1. [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole) にアクセスします
2. Google アカウントでログインします（2 段階認証必須）
3. 一回限りの登録料 5 USD をクレジットカードで支払います
4. ディベロッパー名（公開時に表示される名前）を設定します

### 1-2. プライバシーポリシーの公開

Chrome Web Store はプライバシーポリシーの URL 公開を必須としています。本リポジトリの `docs/PRIVACY.md` を以下のいずれかで公開します。

- GitHub Pages を有効にして `docs/` を公開ソースに指定
- リポジトリの `docs/PRIVACY.md` をそのまま GitHub の Web 上 URL（例: `https://github.com/<owner>/<repo>/blob/main/docs/PRIVACY.md`）として登録
- 独自ドメインで HTML 化して公開

### 1-3. ストア提出素材の準備

`store-assets/` 配下に提出用素材を用意します。

| 素材               | サイズ・形式                   | 必須                                   |
| ------------------ | ------------------------------ | -------------------------------------- |
| アイコン           | 128×128 PNG                    | 必須（`src/icons/icon128.png` と兼用） |
| 小タイル画像       | 440×280 PNG                    | 必須                                   |
| スクリーンショット | 1280×800 PNG（または 640×400） | 1〜5 枚必須                            |
| 大タイル画像       | 920×680 PNG                    | 任意                                   |
| マーキー画像       | 1400×560 PNG                   | 任意（注目アプリ枠）                   |

スクリーンショットは「設定画面」「動作中のオーバーレイ」「ポップアップ UI」の 3 枚を最低限揃えると審査が通りやすいです。

## 2. 初回公開手順

### 2-1. パッケージング

```bash
docker compose run --rm dev mise run lint
docker compose run --rm dev mise run package
```

`dist/x-off-vX.Y.Z.zip` が生成されます。

### 2-2. ストア登録

1. [Developer Dashboard](https://chrome.google.com/webstore/devconsole) で「新しいアイテム」を作成
2. 生成した zip をアップロード
3. 「ストア掲載情報」タブを埋めます
   - 説明文: `store-assets/store-description.md` の本文をコピー
   - 言語ごとの掲載: 日本語と英語の両方を登録
   - スクリーンショット 1〜5 枚を `store-assets/screenshots/` からアップロード
   - 小タイル画像をアップロード
   - カテゴリ: 「生産性」
   - 言語: 日本語 / 英語
4. 「プライバシーの取り扱い」タブを埋めます
   - 単一目的の説明: 「X での閲覧時間を制限し、ユーザーが集中して時間を管理できるようにする」
   - permission justification:
     - `storage`: ユーザーが設定した上限分数と現在の経過秒数を保存するため
     - host permissions (`https://x.com/*`, `https://twitter.com/*`): 計測およびオーバーレイ表示のスクリプト注入と、最後の X タブが閉じた瞬間の検出のため
   - データの収集: 「収集なし」を選択
   - リモートコードの使用: 「いいえ」を選択
5. 「配布」タブを埋めます
   - 公開範囲: 初回は「限定公開（リンクを知っている人のみ）」を推奨。動作確認後に「公開」へ変更します
   - 国・地域: 全世界
6. 「審査の送信」ボタンを押す

### 2-3. 審査と公開

- 審査期間は通常数日〜2 週間です
- 却下された場合、メールで理由が通知されます。指摘箇所を修正して再提出してください
- 限定公開の URL を確認し、別ブラウザプロファイルや別アカウントから実機インストールテストを行ってください
- 問題がなければ Developer Dashboard で公開範囲を「公開」に変更します

## 3. 更新リリース手順

### 3-1. バージョンアップ

1. `src/manifest.json` の `version` をインクリメント（パッチなら `0.1.0` → `0.1.1`）
2. `CHANGELOG.md` に新セクションを追加し、変更内容を記載

### 3-2. パッケージング

```bash
docker compose run --rm dev mise run lint
docker compose run --rm dev mise run package
```

### 3-3. 提出

1. Developer Dashboard で当該アイテムを開く
2. 「パッケージ」タブで新しい zip をアップロード
3. 「審査用パッケージ」と表示されることを確認
4. 必要であればストア掲載情報・スクリーンショットも更新
5. 「審査の送信」ボタンを押す

更新審査は初回より速く完了することが多いですが、permission を増やした場合は再度厳しく審査されます。

## 4. リリース後のチェック

- [ ] Chrome Web Store の本拡張ページで最新バージョンが表示される
- [ ] 「Chrome に追加」ボタンから実機インストールできる
- [ ] インストール直後にオプションページが開く
- [ ] `docs/TESTING.md` のチェックリストを再実施

## 5. 廃止 / 一時停止

- 一時的に新規インストールを停止したい場合: Developer Dashboard で「公開停止」を選択
- 完全に廃止したい場合: 「廃止」を選択。既存ユーザーの拡張は引き続き動作しますが、新規インストールは不可になります

## 6. トラブルシュート

| 症状                                                  | 対処                                                                                                             |
| ----------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| 審査で「permission の説明が不十分」と却下される       | host permission の説明を「計測スクリプトの注入とタブ close 検出」に絞って記載。permission を増やしてはいけません |
| 「単一目的の説明が不明瞭」と却下される                | description と single purpose を一致させ、副次機能を強調しない                                                   |
| zip アップロード時に「manifest が見つからない」エラー | zip のルートに `manifest.json` が来ているかを確認（`unzip -l` で確認）                                           |
| プライバシーポリシー URL が無効と言われる             | 公開アクセス可能な URL であること、HTTPS であることを確認                                                        |
