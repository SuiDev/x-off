# 手動テストチェックリスト

本ドキュメントは X OFF をリリースする前に必ず実施する手動チェックリストです。自動テストはありません（v1.0 スコープ外）。チェック項目を増減させる場合は、変更を含む PR と同じコミットでこのファイルも更新してください。

実施環境:

- Google Chrome 最新安定版（最低でも 130 以上）
- macOS / Linux / Windows のいずれか
- ホスト OS の言語設定: 日本語と英語の両方で 1 周ずつ
- インターネット接続あり（X / Twitter にアクセスするため）

事前準備: `docker compose run --rm dev mise run lint` が通ること、`docker compose run --rm dev mise run package` で `dist/x-off-vX.Y.Z.zip` が生成されること。テストでは zip を unpack したフォルダではなく、`src/` を「パッケージ化されていない拡張機能を読み込む」で直接読ませて構いません（manifest 内容は同一です）。

---

## 1. インストール直後の挙動

- [ ] `chrome://extensions` で「デベロッパーモード」をオンにし、`src/` ディレクトリを読み込む
- [ ] 設定ページ（options）が新規タブで自動的に開く
- [ ] 設定ページのタイトル・見出し・説明・ラベルがホスト OS の言語と一致している
- [ ] 上限分数を未設定のまま X (`https://x.com/`) を開いても、オーバーレイは表示されない
- [ ] ツールバーアイコンの popup を開くと「No limit set yet — open the settings page to configure.」（または日本語版）が表示される

## 2. 上限の設定と保存

- [ ] 設定ページで `0` / `601` / 空文字 / 文字列を入力し「保存」を押すと、エラーメッセージが表示され保存されない
- [ ] 設定ページで `30` を入力して保存すると、「保存しました」メッセージが表示される
- [ ] 設定ページを閉じて再度開くと、入力欄に `30` が復元されている
- [ ] popup 上の「上限分数」入力からも上限を変更でき、保存後に経過/上限/残り表示が更新される

## 3. アクティブ閲覧時間の計測

セットアップ: 上限を `2` 分にして、X のタイムラインを開く。

- [ ] X のタブをフォアグラウンドかつフォーカス状態で 60 秒待つと、popup の経過時間が約 1:00 まで進む（±数秒は許容）
- [ ] 同じタブで他のウィンドウにフォーカスを移すと、経過時間が止まる（popup を閉じて再度開いて確認）
- [ ] X のタブをバックグラウンドにする（別タブをアクティブにする）と、経過時間が止まる
- [ ] X のタブでウィンドウを最小化すると、経過時間が止まる
- [ ] フォアグラウンド + フォーカス状態に戻すと、計測が再開する
- [ ] popup を開いている間は、popup を閉じなくても経過時間がリアルタイムに更新される（これは port 接続による live 表示）

## 4. 上限到達時のオーバーレイ

セットアップ: 上限を `1` 分にして、X のタイムラインを開きフォアグラウンドで 60 秒待つ。

- [ ] フルスクリーンオーバーレイが表示される
- [ ] オーバーレイのタイトル・本文・経過時間表示が正しい言語で表示される
- [ ] マウスホイールでスクロールできない
- [ ] タッチパッドの 2 本指スクロールができない
- [ ] 矢印キー / Space / PageDown / PageUp / Home / End でスクロールできない
- [ ] オーバーレイ表示中もブラウザのタブ切り替え・URL バー操作は可能（拡張は X タブ内のスクロールのみ止める）
- [ ] 別ページ（`https://example.com/` など）にナビゲートするとオーバーレイは表示されない

## 5. 日次リセット

- [ ] DevTools → Application → Storage → Extension Storage → 当該拡張 → `elapsedDate` を昨日の日付（`YYYY-MM-DD`）に書き換える
- [ ] X のタブをリロードすると、`elapsedSec` が 0 にリセットされ、オーバーレイが解除される
- [ ] popup を開くと、経過時間が 0:00、残り時間が上限値そのままになっている

## 6. popup の操作

セットアップ: 上限 `30` 分、経過時間が数分進んだ状態。

- [ ] popup の「経過時間をリセット」ボタンを押すと、経過時間が 0:00 に戻り、残り時間が上限値そのままになる
- [ ] popup の「Share my X usage」ボタンを押すと、X タブ上で投稿 intent が開く（新規タブにならず同じタブが遷移する）
- [ ] 非 X タブで popup の Share ボタンを押すと、新規タブで X の投稿 intent が開く
- [ ] `src/config.js` の `TIP_URL` が空文字の場合、popup の「Support the developer」セクションが非表示になる
- [ ] `TIP_URL` に値が入っている場合、ボタンを押すと当該 URL が新規タブで開く

## 7. 多言語

- [ ] Chrome の言語設定を日本語にした状態で拡張をリロードすると、popup / options / オーバーレイの全文言が日本語になる
- [ ] Chrome の言語設定を英語にした状態で拡張をリロードすると、全文言が英語になる
- [ ] manifest の `name` / `description` が `chrome://extensions` 上で正しく i18n されている
- [ ] Chrome Web Store プレビュー（提出後）でも `name` / `description` が i18n される

## 8. ホスト一致と除外

- [ ] `https://twitter.com/home` でも X と同じ計測・オーバーレイ動作になる
- [ ] `https://x.com/intent/tweet` などの intent ページでは content script が動かない（`exclude_matches` で除外されている）。具体的には、intent ページではオーバーレイが出ず、popup の経過時間も intent タブを開いている間は止まる
- [ ] `https://example.com/` などの他サイトでは一切何も起きない（content script が注入されない）

## 9. service worker / background

- [ ] `chrome://extensions` の「Service Worker (inspect)」を開いてエラーがない
- [ ] service worker を一度 stop した後、X タブをリロードすると content script からの `ping` が正常に応答され、計測が継続する
- [ ] エラーログ（赤）が console に出ていない（`[xsl]` プレフィックスの warn は許容）

## 10. パッケージング

- [ ] `docker compose run --rm dev mise run release` が成功する
- [ ] `dist/x-off-vX.Y.Z.zip` がリポジトリのバージョンと一致する名前で生成される
- [ ] zip を unzip した直後のルートに `manifest.json` がある（サブディレクトリに入っていない）
- [ ] zip 内に `__MACOSX/` や `.DS_Store` が含まれていない
- [ ] zip サイズが妥当（数百 KB 程度。1MB を大きく超えていたら不要ファイルが混入していないか確認）

## 11. クリーンインストール再現

リリース直前のサニティチェック。

- [ ] `chrome://extensions` から拡張を一旦削除する
- [ ] zip を unpack して新規に「パッケージ化されていない拡張機能を読み込む」で読ませる
- [ ] §1〜§4 を再度通す（インストール直後 → 上限設定 → 計測 → 上限到達）

## 不合格時の扱い

1 つでも失敗したら原因を特定して修正し、修正コミット後に **§1 から再度通す**。差分 push の場合でもチェックリストの一部だけ走らせるのではなく、全項目を通すこと。
